    ATTENZIONE

    Da leggere attentamente:


    I controlli hack sono l'ultima risorsa per assicurarsi se il giocatore sta utilizzando metodi illeciti per ottenere vantaggi in gioco, pertanto invitiamo agli amministratori dei servers a non abusare con questo metodo. Inoltre consigliamo di avvalersi di personale competente (Meglio se maggiorenne) e che ha familiarit con i sistemi operativi di tipo:
    WindowsNT/Unix-Like.

    Lo staffer che esegue un controllo deve inoltre essere consapevole di eventuali azioni illegali dovuti a danneggiamento o perdita irreversibile di dati durante un controllo, violazioni della privacy nel caso si controlli cronologia o cartelle non inerenti al videogioco. Bisogna controllare solo le cartelle inerenti a minecraft o se si sospettano injection solo cartelle inerenti al sistema operativo e di norma non permesso allo staffer la visualizzazione di documenti privati.

    Un controllo deve essere eseguito SOLAMENTE a seguito di una segnalazione da parte di utenti multipli, e da parte di un'anticheat professionale. Qualsiasi tipo di abuso o controlli a manetta non saranno visti di buon occhio dagli utenti (e questo influirebbe negativamente l'intera reputazione del server).

    È strettamente consigliato agli utenti di registrare in tutti i controlli hack qualora vi risorgano danni causati dall'incompetenza da parte dello staff in maniera tale da avere una prova per avviare eventuali azioni legali, o semplicemente (se il danno è di gravità minore) avviare una discussione con lo staff del server che ti ospita.

    Gli amministratori dei servers devono assumersi la piena responsabilit qualora il personale commetta reati o abusi di qualsiasi genere con i software di controllo remoto nei confronti dei propri utenti.

    Cos' un SS?

    Un ss o Controllo Hack un controllo che effettua uno staffer ad un player di non presenza di Cheats
    Esistono diversi tipi di hack installabili su Minecraft, le pi famose sono le Ghost e Injections (External/Internal).
    Queste sono dei client particolari che possono essere nascoste facilmente se usate.
    In questa guida ti insegneremo come effettuare un semplice controllo hack.

  • SSTools

    Ecco alcuni SSTools che possono esserti utili.

    LastActivityView

    LastActivityView un tool molto importante.
    LastActivityView uno strumento per il sistema operativo Windows che raccoglie informazioni da varie fonti su un sistema in esecuzione e visualizza un registro delle azioni eseguite dall'utente e degli eventi verificatisi su questo Computador. L'attivit visualizzata da LastActivityView comprende: Esecuzione di file .exe, Finestra di dialogo Apri / Salva, Apri file / cartella da Explorer o altro software, installazione software, arresto / avvio del sistema, arresto dell'applicazione o del sistema, connessione / disconnessione della rete e altro.

    Scaricalo




    USBdeview

    USBDeview un'utility compatta che si incarica di mostrare l'elenco dei dispositivi USB collegati al pc cos come quelli connessi in precedenza. Per stilare l'elenco, USBDeview attinge alle informazioni conservate nel registro di sistema di Windows. Per ciascuna periferica USB, il programma riporta nome, descrizione, tipologia del dispositivo, la data del primo utilizzo, la data del pi recente collegamento al sistema, il numero seriale, il produttore e cos via..

    Scaricalo




    Process Hacker 2

    Inanzi tutto Process Hacker 2 un tool per le versioni a 32 e a 64 bit di Windows che permette di manipolare i processi in maniera molto sofisticata. Il programma ha tre pannelli di visualizzazione: processi, servizi e network. I processi sono visualizzati in modo gerarchico e sono evidenziati con codici di colore. Le funzione di terminazione dei processi riescono a bypassare varie protezioni e include quelle di alcuni malware e rootkit.

    Scaricalo




    UserAssistView

    Questa utility decodifica e visualizza l'elenco di tutte le voci UserAssist memorizzate nella chiave HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ Explorer \ UserAssist nel registro. La chiave UserAssist contiene informazioni sui file exe e sui collegamenti aperti di frequente. possibile salvare l'elenco delle voci UserAssist in file di testo / html / xml / csv, nonch eliminare elementi indesiderati.

    Scaricalo




    RegScanner

    RegScanner una piccola utility che consente di eseguire la scansione del registro di sistema, trovare i valori del registro di sistema desiderati che corrispondono ai criteri di ricerca specificati e visualizzarli in un elenco. Dopo aver trovato i valori del registro di sistema, possibile passare facilmente al giusto valore in RegEdit, semplicemente facendo doppio clic sull'elemento del registro di sistema desiderato. inoltre possibile esportare i valori del registro di sistema trovati in un file reg che pu essere utilizzato in RegEdit..

    Scaricalo




    Search Everything

    Search Everything un'utilit di ricerca desktop per Windows in grado di trovare rapidamente file e cartelle per nome. Da gennaio 2013, tutto stato sviluppato attivamente dopo una pausa di quattro anni. Questa utility rilasciata con una licenza per software libero che consente modifiche e ridistribuzioni commerciali, richiedendo solo l'attribuzione.
    Cartelle specifiche su qualsiasi file system possono anche essere aggiunte all'indice, ma l'indicizzazione delle cartelle che non utilizzano NTFS o ReFS sar lenta, anche se la ricerca con l'indice completato non lo sar. Indipendentemente dal file system utilizzato su unit e cartelle indicizzate, il software cerca nel suo indice i nomi dei file corrispondenti a un'espressione di ricerca dell'utente, che pu essere un frammento del nome del file di destinazione o un'espressione regolare, visualizzando risultati intermedi e immediati come ricerca il termine stato inserito.

    Scaricalo




    OpenSavedFiles

    OpenSavedFiles un semplice strumento che visualizza l'elenco dei file precedentemente aperti con la finestra di dialogo standard di apertura / salvataggio di Windows. Per ogni file nell'elenco, vengono visualizzate le seguenti informazioni: Nome file, Estensione, Ordine (L'ordine in cui i file sono stati aperti per ogni estensione), Open Time (Disponibile solo per l'ultimo file aperto di ogni tipo di file), File modificato / Tempo di creazione, dimensione del file e attributi del file.

    Scaricalo




    WinPrefetchView

    Ogni volta che si esegue un'applicazione nel proprio sistema, viene creato un file Prefetch che contiene informazioni sui file caricati dall'applicazione dal sistema operativo Windows. Le informazioni nel file Prefetch vengono utilizzate per ottimizzare il tempo di caricamento dell'applicazione al successivo avvio. WinPrefetchView una piccola utility che legge i file Prefetch memorizzati nel tuo sistema e visualizza le informazioni in essi memorizzate. Guardando in questi file, puoi sapere quali file utilizza ogni applicazione e quali file vengono caricati all'avvio di Windows.

    Scaricalo

    CMD (Prompt dei comandi)

    Il Prompt dei Comandi uno strumento che funziona da riga di comando, disponibile nella maggior parte dei sistemi operativi Windows, che viene utilizzato per eseguire dei comandi immessi dall'utente. Con l'uso di questi comandi possibile automatizzare delle attivit tramite script e file batch, eseguire funzioni amministrative avanzate e risolvere determinati tipi di problemi di Windows. Il Promt dei Comandi non semplicemente una metodologia antiquata per usare Windows, ma uno strumento vero e proprio, molto potente, che permette l'uso di tecniche avanzate per controllare e gestire al meglio il proprio Computador.
    Il prompt dei comandi (chiamato anche cmd.exe, diretto successore del vecchio command.com) un programma interno a Microsoft Windows avente un'interfaccia grafica stile MSDOS (Microsoft Disk Operating System). ufficialmente denominato processore dei comandi di Windows (Infatti rappresenta un'emulatore del vecchissimo MS-DOS), ma a volte viene anche chiamato shell dei comandi o prompt cmd, o facendo riferimento al gi citato nome del file che lo rappresenta (cmd.exe).
    Eseguendo il prompt, Windows apre una finestra (con sfondo nero) dove possibile digitare dei comandi da fare interpretare al sistema operativo. Il termine Prompt dei comandi si riferisce anche alla parentesi angolare chiusa (>, coincidente con il segno di maggiore) che indica che linterfaccia della riga di comando pronta per accettare comandi.

    Nel CMD ci sono dei comandi specifici.
    Una volta aperto scrivere: TaskList
    Un'altro comando utile il comando Tree/f. Per cercare Clicker basta premere, Ctrl+F, e scrivere il nome del clicker che cercate.
    Se si ha gi utilizzato Process Hacker 2 o PowerShell questo procedimento un po inutile.

    Il Cestino

    Alcuni utenti prima di essere controllati eliminano dei file importati e se succede potete procedere al ban immediato dell'utente.
    Per vedere l'ultima modifica del cestino premere nella tastira: Tasto Windows + r; si aprir il Searcher "Esegui" e scrivere C:\$Recycle.bin dopodich aprir un file con l'ultima modifica del cestino.
    Se l'utente ha modificato l'ora prima del controllo possibile sgamarlo con: Windows + R -> Eventvws -> Sistema -> Id 1. e guardate il log pi recente.

    [Facoltativo] Residui autoclickers, cestino o shift + canc

    Molti utenti che eliminano il file autoclicker spesso si dimenticano di eliminare le dipendenze presenti in: %temp%
    Cercate in quella special folder utente li seguenti stringhe: Jnative / JnativeHook. Qualora risultasse positiva la ricerca si pu bannare tranquillamente.
    Nota: Le seguenti stringhe di ricerca di residui di autoclickers o altri dll sospetti possono essere usati su process hackers se si ha dimestichezza con quest'ultimo.
    Se un giocatore elimina i file di cheats, utilizzando il cestino o shift + canc, (quasi) sempre vincolato da qualche residuo e dall'eliminazione non totale del file. Difatti, andando in "C:\Users\NomeUtente\AppData\Local" e cercando %temp%, si potranno trovare i file usati nella giornata e nelle giornate precedenti, sempre buona abitudine controllare la data.

    [Spiegazione] La dipendenza: JnativeHook

    JNativeHook una libreria per fornire listeners globali di tastiera e mouse per Java. Questo ti permetter di mettere in comunicazione scorciatoie globali o movimenti del mouse che altrimenti sarebbero impossibili usando Java puro. Per eseguire questa attivit, JNativeHook sfrutta il codice nativo dipendente dalla piattaforma attraverso l'interfaccia nativa di Java per creare hook di basso livello a livello di sistema e consegnare tali eventi all'applicazione.

    Tools:



    LastActivityView:


    Il programma non richiede alcun processo di installazione o file DLL aggiuntivi. Dopo aver eseguito LastActivityView, esegue la scansione del Computador e visualizza tutte le azioni e gli eventi trovati sul sistema. possibile selezionare uno o pi elementi e salvarli in file xml / html / csv / delimitati da tabulazioni (Ctrl + S) o copiarli negli Appunti (Ctrl + C), quindi incollare i dati in Excel o in altri software.

    UsbDeview:


    Il programma non richiede alcun processo di installazione o file DLL aggiuntivi. Basta copiare il file eseguibile (USBDeview.exe) in qualsiasi cartella ti piace ed eseguirlo. La finestra principale di USBDeview mostra tutti i dispositivi USB installati sul sistema. possibile selezionare uno o pi elementi, quindi disconnetterli (scollegarli), disinstallarli o semplicemente salvare le informazioni nel file text / xml / html.

    UserAssistView:


    Il programma non richiede alcun processo di installazione o file DLL aggiuntivi. Per iniziare a usarlo, sufficiente eseguire il file eseguibile - UserAssistView.exe Dopo averlo eseguito, la finestra principale di UserAssistView visualizza l'elenco di tutti gli elementi UserAssist memorizzati nel registro. possibile selezionare uno o pi elementi, quindi trasformarli in un file o eliminarli.

    RegScanner:


    Il programma un eseguibile autonomo e non richiede alcun processo di installazione o DLL aggiuntive. Per iniziare a usarlo, basta eseguire il fileecutbale - regscanner.exe Dopo aver eseguito RegScanner, necessario selezionare le opzioni di scansione / ricerca desiderate e fare clic su OK per avviare la scansione.

    SearchEverything:


    Il programma pu essere avviato sia come eseguibile autonomo, sia come installatore. Per operare bisogna semplicemente andare nella casella di ricerca, inserire la stringa desiderata e premere invio.

    OpenSavedFiles:


    Il programma non richiede alcun processo di installazione o file dll aggiuntivi. Per iniziare a usarlo, sufficiente eseguire il file eseguibile - OpenSaveFilesView.exe Dopo aver eseguito OpenSaveFilesView, la finestra principale visualizza l'elenco dei file precedentemente selezionati con la finestra di dialogo standard di apertura / salvataggio di Windows.

    WinPrefetchView:


    Il programma non richiede alcun processo di installazione o file DLL aggiuntivi. Per iniziare a usarlo, sufficiente eseguire il file eseguibile - WinPrefetchView.exe La finestra principale di WinPrefetchView contiene 2 riquadri: il riquadro superiore visualizza l'elenco di tutti i file Prefetch nel sistema. Quando si seleziona un file nel riquadro superiore, nel riquadro inferiore viene visualizzato l'elenco dei file archiviati all'interno del file Prefetch selezionato, che rappresentano i file caricati dall'applicazione nei tempi precedenti in cui stato utilizzato. Questo anche un file di prefetch speciale, con il nome del file 'NTOSBOOT-B00DFAAD.pf', che pu mostrare l'elenco dei file che vengono caricati durante il processo di avvio di Windows.

    Per maggiori informazioni bisogna andare nella sezione tools.

    [Directory] Elenco completo delle shells: /etc/shells

    [Terminale] Tutti i files eseguiti dal processo: lsof -p [ProcessID] | grep [ProcessName]

    [Directory] Locazione di tutti i logs e tutti gli eventi di sistema + file temporanei (Ultimi binari eseguiti/usb etc..): /var/log | /var/tmp

    [History] Cronologia comandi terminale premere i tasti direzione: UP - DOWN

    [Directory] Locazione cestino (MacOS): ~/.Trash

    [Directory] Locazione cestino (Linux): $HOME/.Trash | $HOME/.local/share/Trash | $HOME/Desktop/.Trash
